Built around accessibility, not bolted on

QUILL's design rules are non-negotiable, and they shape everything from the editor surface to the extension platform.

Spoken-first

Every command reports a clear, consistent outcome with NVDA, JAWS, and Narrator parity. A silent success is treated as a bug.

Keyboard-complete

Everything is reachable and operable from the keyboard alone — no mouse-only affordances, no focus traps.

Stock-control editor

The writing surface uses standard, accessible controls and stays plain-text-first, so it behaves the way assistive tech expects.

No silent network

Every cloud or AI action is explicit, per-action opt-in, with visible progress and an announced result. Your document never leaves the device without consent.

Robust by default

Schema-validated stores, atomic writes, and recovery backups keep your work safe.

Extensible & sandboxed

Quillins add commands, snippets, menus, and hotkeys — running in a capability-gated sandbox, never with silent access to your files or network.

The Quillin platform — extend QUILL your way

A Quillin is a sandboxed extension for QUILL. You can build one with a JSON file and a few lines of Python — no build tools, no compiled code. Quillins integrate seamlessly into QUILL's menus, keyboard shortcuts, and screen reader announcement engine, so they are indistinguishable from built-in features at the point of use.

Two ways to build: a JSON-only manifest (Layer 1) for snippets and shortcuts, or a Python handler (Layer 2) for real logic that runs safely in its own process.

Layer 1 — Snippets and shortcuts

Write a single JSON manifest to add menu items, hotkeys, and text snippets that expand placeholders like ${selection}, ${date}, and ${clipboard}. No Python required.

Layer 2 — Python handlers

Add a Python entry module for genuine custom behaviour. Your code runs in a sandboxed subprocess and only accesses what it explicitly declared in the manifest. A crash in your Quillin cannot affect the editor or your document.

What a Quillin can do — every capability is declared in the manifest and disclosed to you at install time. No surprises.

Read and write your document

A Quillin can read document text, the current selection, and the cursor position, then insert or replace text as an undoable edit. All edits pass through the same undo history as your own typing.

Speak through the screen reader

Any Quillin command can announce its outcome through QUILL's announcement engine with full NVDA, JAWS, and Narrator parity — the same engine built-in commands use. A silent Quillin is treated as a bug.

Add menus, commands, and hotkeys

Quillins attach their commands to any top-level menu and to the right-click context menu. Keyboard shortcuts use QUILL's existing binding grammar. Conflicts are detected and reported, never silently overridden.

Read and write files (with your permission)

With the filesystem capability, a Quillin can read notes from a folder or write output to a file. You are informed at install time and file access never happens silently.

Connect to the web (with your permission)

With the network capability, a Quillin can call a web API, trigger a webhook, or fetch content. Every outbound request requires per-action consent — you see exactly where your data is going before it leaves your device.

Remember your preferences

The storage capability gives a Quillin a private key-value store. Settings you configure in one session are remembered in the next, without touching the filesystem directly.

Show choice prompts and status messages

A Quillin can present a native, accessible choice dialog so you pick from a list, and can post progress updates to the status bar — no custom dialogs, no visual-only feedback.

Read and write the clipboard

A Quillin can paste content from the clipboard or copy its output to it, so you can chain QUILL work with other apps in your workflow.

See what's possible — showcase Quillins

These four showcase Quillins demonstrate the range of what the platform can do. Each is a complete, working example with full source code and documentation.

Accessibility Auditor

Scans your document for common accessibility problems: heading levels that skip, image markers without descriptions, and punctuation patterns that create screen reader noise. One command in the Tools menu produces an itemized report inserted at the cursor, with the issue count spoken immediately.

Reads document, inserts report, announces result — no network, no files.

View source and documentation →

Ghostwriter's Mirror

Transforms your selected text into a different voice persona with one menu action. Choose Legal Formalist to shift casual writing toward formal, precise language, or Victorian Poet for dramatic, romanticized phrasing. The selection is replaced as a single undoable edit; Ctrl+Z restores the original.

Reads selection, replaces selection, announces result — no network, no files.

View source and documentation →

Research Alchemist

Siphons bullet points and key facts from every text and Markdown file in a folder, tags each item with its source file and line number, and inserts the compiled list into your document. Drop your notes into a designated folder, run one command, and your research is assembled and attributed.

Reads files from a folder you designate (disclosed at install time), inserts into document.

View source and documentation →

Zen Focus Bridge

Marks a focused writing session in your document and bridges to your broader environment — launching focus music, silencing notifications, or calling any HTTP endpoint — with a single command from the View menu. Activate and deactivate by keyboard; the screen reader announces each transition.

Inserts session markers, makes network calls with per-action consent.

View source and documentation →
