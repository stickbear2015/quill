# QUILL Accessibility Fix Log

Companion to x2.md. Each completed fix is recorded here with its solution and impact.
Items are moved from x2.md to this file as they are resolved.

## #3 — TS console opens with Python selected

**Solution:** Added `set_language(lang: str)` public method to `ConsoleWindow` that sets
the language Choice control and re-labels the prompt. `open_typescript_console()` in
`main_frame_devtools.py` calls `win.set_language("TypeScript")` before showing the window.

**Impact:** TypeScript console now launches in TypeScript mode. Previously, typed TS code
executed as Python and threw syntax errors with no obvious explanation.

## #4 (partial) — Up/Down history arrows fire mid-edit of multi-line input

**Solution:** `_on_key_down` in `ConsoleWindow` now calls `_caret_on_first_line()` /
`_caret_on_last_line()` before triggering history navigation. Arrows within a multi-line
block scroll within the input rather than replacing it with a history entry.

**Impact:** Eliminates accidental data loss when a user arrows to review what they typed.
Screen reader users can navigate multi-line input without it being replaced.

## #7 — Clipboard announcements were truthful-only when open succeeded

**Solution:** `_copy_transcript()` in `ConsoleWindow` now checks the return value of
`wx.TheClipboard.Open()`. On success it announces "Transcript copied to clipboard". On
failure it announces "Could not open clipboard". Previously it announced nothing at all.

**Impact:** Screen reader users now hear the outcome of the copy action.

## #10 — QDC keyboard shortcuts only active when input TextCtrl had focus

**Solution:** `ConsoleWindow` binds `EVT_CHAR_HOOK` at the wx.Frame level.
`_on_char_hook` handles Esc (close), F1 (help), Ctrl+L (clear), Ctrl+Shift+C (copy
transcript), Ctrl+S (save transcript) regardless of which child control has focus.

**Impact:** Keyboard-only and screen reader users can close/control the console from
any focused element inside it (transcript, buttons, language selector).

## #11 — Esc/close returned focus to main wx.Frame, not the editor

**Solution:** `ConsoleWindow` accepts a `focus_editor_cb` parameter. `_return_focus()`
calls the callback if provided, otherwise falls back to `_parent.SetFocus()`.
`main_frame_devtools._dt_focus_editor()` does `self.editor.SetFocus()`.

**Impact:** Closing the console returns caret to the document editor, matching the PRD
and expected screen reader flow.

## #51 — No sys.excepthook; crashes silently closed QUILL with zero feedback

**Solution:** Added `_install_excepthook()` to `quill/__main__.py`, called immediately
after `setup_fault_handler()`. The handler:
1. Writes a timestamped crash report to `<app-data>/crash-reports/crash-<ts>.txt`.
2. Shows `ctypes.windll.user32.MessageBoxW` (Win32 native) with the error and file path.
   This dialog is read by Narrator/NVDA even with no wx running.
3. Calls `sys.__excepthook__` to preserve the default traceback on stderr.

**Impact:** Any unhandled exception now produces a spoken, readable error dialog telling
the user what crashed and where the report is. Previously a crash was completely silent.

## #52 — prism_bridge TTS blocked the UI thread (runAndWait on UI thread)

**Solution:** Added a dedicated daemon thread `quill-tts-worker` with a `queue.Queue`.
`announce()` in TTS-fallback mode calls `_tts_queue.put_nowait(message)` and returns
immediately. The worker calls `engine.say()` + `engine.runAndWait()` off the UI thread.
`task_done()` calls in the worker loop enable `flush_tts_for_tests()` for test isolation.

**Impact:** A two-sentence TTS announcement no longer freezes the QUILL UI for ~5 seconds
for low-vision users relying on self-voicing. The UI thread returns immediately.

## #33 — GitHub URL parser rejected query strings, anchors, and encoded paths

**Solution:** Replaced the manual string-split parser in `_parse_github_blob_url` with
`urllib.parse.urlparse` + `urllib.parse.unquote`. Query strings (?plain=1) and line
anchors (#L10) are stripped automatically. Percent-encoded path characters are decoded.
The host check uses `parsed.netloc == "github.com"` so any URL structure still works.

**Impact:** Copying a URL from the GitHub "Copy permalink" button or from the browser
address bar now opens correctly regardless of query params or line anchor.

## #37 — Vendor autoupdate _version_tuple mapped pre-release versions to (0,)

**Solution:** Replaced `int(x)` per segment (which throws ValueError on "1.2.3rc1") with
`re.match(r"(\d+)", segment)` — extracts the leading numeric part of each segment, falls
back to 0 only for fully non-numeric segments. Empty input still returns (0,).

**Impact:** A server version like "1.2.3rc1" is now parsed as (1,2,3) rather than (0,),
preventing a typo in the version field from silently disabling updates for all users.

## #1 — ts_console._handle_invoke ran wx widget calls on the Node reader thread

**Solution:** Added a `ui_call: Callable[[Callable[[], Any]], Any] | None` parameter to
`TypeScriptConsole.__init__`. When set, `_dispatch` marshals each invoke execution
through it — blocking the reader thread on a `threading.Event` until the UI thread
completes the call. In `DevToolsMixin._dt_ts_console()`, the console is constructed
with `ui_call=self._dt_marshal_to_ui`. `_dt_marshal_to_ui` posts a wrapper to
`wx.CallAfter`, waits on an `Event`, and re-raises any exception. The UI thread
is free during invoke processing because `execute()` runs on `_dt_ts_execute_thread`,
not the UI thread.

**Impact:** Eliminates undefined behavior: all `ConsoleHost` wx widget access
(`editor.GetValue()`, `WriteText`, etc.) now runs on the UI thread as required.
No deadlock risk — the design was already correct (execute on daemon thread,
UI thread free), just missing the marshal step.

## #23 — github-temp filename collision caused wrong-repo commits

**Solution:** In `_on_file_fetched`, replaced:
```python
local_path = temp_dir / Path(remote_file.path).name
```
with a content-addressed subdirectory:
```python
slot_key = f"{remote_file.repository.full_name}:{remote_file.ref}:{remote_file.path}"
slot_hash = hashlib.sha256(slot_key.encode()).hexdigest()[:16]
slot_dir = temp_dir / slot_hash
local_path = slot_dir / Path(remote_file.path).name
```
Each unique `(repo, ref, path)` combination now lands in its own slot, so the
`origins` dict entry is never overwritten by a same-named file from a different repo.

**Impact:** Opening `README.md` from two different repositories no longer causes
Tab A's "Save to GitHub" to silently commit to Tab B's origin. The collision that
could corrupt data across tabs is eliminated.
